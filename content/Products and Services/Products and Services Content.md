# 1. Current Strategic View

## Where Stratum is today

Stratum already provides real client value through managed support, onsite service, client documentation, server and network maintenance, software configuration, incident response, backup and recovery, proactive monitoring, vendor coordination, consulting, and project work.

## Where Stratum is going

Stratum is moving from a mixed reactive MSP model toward a more structured managed service, security, resilience, and executive advisory model.

The goal is not to become a bundle of tools. The goal is to become a clearer operating partner that provides:

- dependable managed support
- stronger security and recovery posture
- cleaner documentation and accountability
- clearer package structure
- more strategic ownership over time

## What has changed in the thinking

The product and service system is no longer being shaped only by broad service categories.

The newer model is based on:

- package architecture
- product economics
- customer outcomes
- environment complexity
- usage patterns
- advisory depth

That means this page now serves as the bridge between the brand story and the actual package model.

# 3. The Package Architecture

## Core shift

Stratum’s package line should now be understood through **customer usage and environment complexity first**, not user count first.

Users still matter, but they are only one input in the pricing and package decision model.

The better model is:

- support usage
- managed endpoints
- servers
- backups
- network / site complexity
- licenses and hard costs
- security / risk needs
- account and advisory needs

## Customer-facing package line

| Package | Main Purpose | Best Fit |
| --- | --- | --- |
| Stratum Essentials | Managed foundation | Low-complexity clients needing structure, support, endpoint protection, and documentation. |
| Stratum Business | Managed operations | Active businesses with recurring support demand, managed endpoints, network/server awareness, and proactive maintenance needs. |
| Stratum Secure | Managed operations + security posture | Clients needing cybersecurity controls, backup posture, security awareness, risk review, and cyber-insurance support. |
| Stratum Complete | Full technology partnership | Clients needing reporting, roadmap, lifecycle planning, budgeting, security audits, and strategic advisory. |

## Core package principle

Complete is not “more tools.” Complete is **more ownership**.

The package system should make Stratum easier to buy from, easier to understand, and easier to manage internally.
