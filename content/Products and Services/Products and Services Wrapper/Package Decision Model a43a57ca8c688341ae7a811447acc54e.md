# Package Decision Model

## Purpose

This page explains how Stratum should decide package fit.

The package choice should not come from user count alone. It should come from the full operating picture.

## Core rule

Choose the package from:

- support usage
- managed endpoints
- servers
- backups
- network / site complexity
- licenses and hard costs
- security / risk needs
- account and advisory needs

## Assessment areas

| Assessment Area | What We Measure | Why It Matters |
| --- | --- | --- |
| Support usage | Monthly hours, ticket volume, work orders | Shows real labour demand |
| Managed endpoints | Desktops, laptops, shared workstations | Shows device and tooling responsibility |
| Servers | Physical and virtual servers, server roles | Shows infrastructure complexity |
| Backups | M365, server, workstation, restore testing | Shows recovery responsibility |
| Network / site | Locations, firewall, switches, Wi-Fi, ISP coordination | Shows site complexity |
| Licenses | Microsoft, EDR, DNS, backup, phones, awareness tools | Shows hard monthly cost |
| Security / risk | Cyber insurance, sensitive data, risk exposure | Shows Secure / Complete fit |
| Account needs | Reporting, roadmap, budgeting, lifecycle planning | Shows Complete fit |

## Package fit summary

| Package | Best fit | Main signal |
| --- | --- | --- |
| Stratum Essentials | Low-complexity managed foundation | Low support usage and low environment complexity |
| Stratum Business | Managed operations | Recurring support demand and active device/server environment |
| Stratum Secure | Operations plus security and backup posture | Cyber risk, backup scope, and insurance/security demands |
| Stratum Complete | Full technology partnership | Leadership reporting, planning, governance, and strategic ownership |

## Rule for operations

The package should be chosen from the assessment, then reviewed quarterly or annually so it can be right-sized over time.
