# 3. Gaps Between Today and Where We Want To Go

## Gap 1: We do a lot of the work, but it is not fully packaged

Today, many services are still described as activities.

Examples:

- Troubleshooting
- Onsite support
- Backup checks
- Vendor calls
- Patching
- Documentation
- Roadmap conversations

The customer should not only see activities. They should see clear service lines and outcomes.

### What needs to be built

- Named service plans
- Clear inclusions and exclusions
- Pricing structure
- Internal delivery checklist
- Client-facing service descriptions
- Website service pages

---

## Gap 2: Cybersecurity exists, but it is not yet a complete product line

We already respond to security issues and have cybersecurity capability, but we need a more defined offer.

### Current state

- Security incidents are handled
- Alerts are reviewed
- Contractor or specialist help exists
- Security is part of support

### Needed future state

- Clear cybersecurity packages
- Ownership of the security service line
- Defined tools and standards
- Reporting
- Cyber insurance readiness process
- Security maturity roadmap

---

## Gap 3: Backup work exists, but recovery assurance is not yet clearly sold

Customers may assume they are protected because backups exist.

That is not enough.

### Current state

- Backup checks
- Test restores
- Backup troubleshooting
- Recovery work when needed

### Needed future state

- Backup assurance plan
- Scheduled restore testing
- Recovery documentation
- Microsoft 365 backup clarity
- Business continuity planning
- Regular customer reporting

---

## Gap 4: Account management needs to become a real operating rhythm

This is one of the biggest growth opportunities.

We need a dedicated way to stay connected to customers, review their environment, explain risk, and plan future work.

### Current state

- Customer communication happens around tickets and projects
- Some strategy conversations happen
- Service Manager and technicians carry a lot of the relationship

### Needed future state

- Account Manager role
- Monthly customer reports
- Regular check-ins
- Quarterly reviews
- Technology roadmaps
- Lifecycle planning
- Clear handoff between service, projects, and account management

---

## Gap 5: Projects need a stronger delivery framework

Separating tickets from projects is the right direction.

Now the project side needs its own system.

### Current state

- Project work gets done
- Migrations and upgrades happen
- Skilled techs carry the work

### Needed future state

- Project intake process
- Standard scoping template
- Solutions Architect ownership
- Project schedule
- Change control
- Customer communication plan
- Closeout documentation
- Handoff back to support

---

## Gap 6: Strategy is happening, but it is not yet a named product

We already advise customers.

But if it is not named, packaged, and scheduled, it becomes informal and inconsistent.

### Current state

- Technology reviews
- Roadmap conversations
- Advisory sessions
- Recommendations

### Needed future state

- Technology Strategy service
- Quarterly business review format
- Annual technology planning
- Budget roadmap
- Risk summary
- Lifecycle plan

---

## Gap 7: Business systems, AI, ERP, and CRM need a clearer methodology

This could become a strong future service area, but it needs guardrails.

We need to define what we are actually good at, what we can deliver repeatedly, and what we should avoid taking on too early.

### Current state

- Software and automation capability exists
- CRM/ERP/AI opportunities are being discussed
- Internal systems are being evaluated

### Needed future state

- Defined implementation process
- Discovery checklist
- Business process mapping
- Tool selection framework
- Data and permission review
- Training and adoption plan
- Support model after implementation

---

## Gap 8: Marketing and website need to match the service model

The website should not just describe us as an IT company.

It should clearly show the service categories we want to sell and the outcomes customers care about.

### Current state

- Brand and domain work is underway
- Services are being discussed
- Product structure is still being formed

### Needed future state

- Website pages for each main service line
- Clear product language
- Buyer-focused descriptions
- Industry-specific landing pages
- Service packages tied to Structure, Security, Stability, Simplicity, and Stewardship
